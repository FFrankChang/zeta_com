# 则知官网 Sitemap 与页面内容说明（V2）

> 用途：本文件用于指导 AI Agent 或开发团队完成则知官网的整体信息架构、页面搭建与首版内容组织。  
> 适用阶段：当前官网内容尚不丰富，优先建立清晰、可信、可扩展的 ToB 官网骨架，而不是一次性铺开过多栏目。  
> 当前原则：少而稳，先立住能力体系与案例库，再逐步扩充内容。

---

## 1. 网站定位

则知是一家面向企业客户的 AI Agent 落地服务与技术公司。

则知为企业构建的，不是单一 Agent，也不是孤立场景自动化，而是一套**可编排、可协同、可自迭代、可治理的企业级智能体操作能力体系**。

这套体系让企业能够把**模型、知识、系统、流程与人**连接起来，形成能够**持续执行、持续学习、持续优化**的 Agent 集群网络，最终沉淀为企业的新型生产基础设施。

更锋利一点地说：

> 则知帮助企业获得的，不只是“用几个 AI 工具提效”的能力，而是“组织级部署、调度、监督和进化 AI 劳动力”的能力。

因此，官网的目标不是单纯展示“我们会做 AI”，而是让访客快速理解：

1. 则知是谁，做什么。
2. 则知能为企业构建什么层级的智能体能力。
3. 则知已经做过什么，是否真的能落地。
4. 企业客户如何进一步了解、体验和联系则知。

官网整体结构应围绕以下四个核心目标展开：

- **建立认知**：讲清则知的定位与能力体系。
- **建立信任**：用案例、客户故事、方法和治理表达建立可信度。
- **降低理解门槛**：通过简洁页面结构与可体验案例，让访客快速理解 Agent 的价值。
- **承接转化**：为意向客户提供明确的联系入口。

## 2. 顶部导航建议

建议首版官网顶部导航使用以下一级栏目：

- 首页 Home
- 能力体系 Capabilities
- 案例库 Case Library
- 资源中心 Resources
- 关于则知 About Zeta

右上角建议保留 2 个高价值入口按钮：

- **体验案例**：跳转到“案例库 / 可体验案例”
- **联系我们**：跳转到“关于则知 / 联系我们”

说明：
- 现阶段不单独设置“解决方案”一级栏目，避免内容过薄。
- 现阶段不单独设置“预约演示”一级栏目，体验与转化入口合并在案例库与关于则知中。
- 后续若案例、行业方案和交付内容增多，可再升级出“解决方案”或“交付服务”栏目。

# 5.1 首页 Home

## 页面目标

首页需要在较短时间内完成以下四件事：

1. 讲清则知是一家什么公司。
2. 讲清则知不是在卖单点工具，而是在构建企业级智能体能力体系。
3. 用案例证明则知不是空泛概念。
4. 引导访客进入案例体验或商务联系。

## 推荐模块结构

### 1）首屏 Hero
**目标**：第一屏就讲清核心价值。  
**建议内容**：
- 一句主标题，概括则知的核心价值主张。
- 一句副标题，说明则知构建的是企业级智能体操作能力体系，而不是单一 Agent。
- 两个按钮：体验案例 / 联系我们。
- 右侧可放科技感视觉、3D 抽象图形、Agent 网络图或编排流意象。

**建议主标题方向示例**：
- 帮助企业构建可编排、可协同、可自迭代、可治理的 AI 劳动力体系。
- 让企业真正获得组织级部署、调度、监督和进化 AI 劳动力的能力。
- 把模型、知识、系统、流程与人连接起来，形成持续运行的 Agent 集群网络。

### 2）能力总纲 Capability Thesis
**目标**：先给出高于“功能点”的总定义。  
**建议内容**：
- 一段 2~3 句话的总纲文案。
- 一个简洁结构图，表达“模型 + 知识 + 系统 + 流程 + 人 → Agent 集群网络 → 企业新型生产基础设施”。

### 3）六层能力概览 Six-Layer Snapshot
**目标**：快速给出则知能力地图。  
**建议内容**：
- 6 张能力卡片，每张卡片一句定义 + 一句业务价值。
- 可以采用纵向层级或环形结构展示。

**六层建议名称**：
1. Agent 化封装能力
2. Agent 编排与集群协同能力
3. 业务执行闭环能力
4. 自迭代与自优化能力
5. 人机共管与治理能力
6. 企业级智能运营能力

### 4）精选案例 Featured Cases
**目标**：快速建立“已经做过”的信任感。  
**建议内容**：
- 2~3 个案例卡片。
- 每个卡片至少包含：案例名称、场景类型、简短成果、跳转按钮。
- 优先放最能体现“编排、执行、治理、闭环”的案例，而不是最简单的问答类案例。

### 5）可体验 Agent Try a Demo
**目标**：降低用户理解门槛。  
**建议内容**：
- 1~3 个简化版体验入口。
- 配简短说明：当前为简化体验版，正式项目支持接入企业知识、系统与流程。

**体验方向示例**：
- 知识助理 Agent
- 销售跟进 Agent
- 数据分析 Agent

### 6）公司简介 About Snapshot
**目标**：补充品牌背景与信任感。  
**建议内容**：
- 简短介绍则知是谁。
- 则知服务对象是谁。
- 为什么则知强调“能力体系”而不是“单点工具”。
- 跳转到“关于则知”。

### 7）页脚 Footer
**目标**：提供基础导航与收口。  
**建议内容**：
- 导航链接
- 联系方式
- 公司信息
- 隐私政策 / 服务条款（后续可补）

# 5.2 能力体系 Capabilities

## 页面目标

这个页面不是单纯罗列功能，而是要清楚回答：

- 则知到底在为企业构建什么能力。
- 这套能力为什么比“几个 AI 工具提效”更高级。
- 这套能力如何落到真实交付。
- 企业为什么可以把它当作长期生产基础设施来建设。

## 页面建议主叙事

建议整页围绕以下总述展开：

> 则知为企业构建的，不是单一 Agent，也不是孤立场景自动化，而是一套可编排、可协同、可自迭代、可治理的企业级智能体操作能力体系。  
> 这套体系让企业能够把模型、知识、系统、流程与人连接起来，形成能够持续执行、持续学习、持续优化的 Agent 集群网络，最终沉淀为企业的新型生产基础设施。

## 推荐模块结构

### 1）能力总纲 Capability Thesis
**内容方向**：
- 先用 1 个核心命题讲清能力方向。
- 再用一张结构图表达从“单点 AI 工具”升级到“组织级 AI 劳动力系统”的差异。

**建议对比表达**：
- 不是给系统加一个聊天框，而是把企业中的认知性工作抽象成可执行 Agent。
- 不是单 Agent automation，而是多 Agent orchestration。
- 不是停留在 Copilot，而是进入 Operational AI。
- 不是一次性交付静态 Demo，而是可持续优化的运行体系。
- 不是黑箱工具，而是可治理、可监督、可审计的生产系统。
- 不是买几个 Agent，而是建立 AI Operating Model。

### 2）六层能力体系 Six-Layer System
**内容方向**：
- 这是能力页的核心模块。
- 每一层都要包含：定义、前沿表达、落地方式、本质。
- 页面表现上可以做成长滚动叙事，也可以做 6 张展开卡片。

#### 第一层：Agent 化封装能力
**定义**：把原本分散在人、文档、系统和流程里的工作单元，封装成可以被调用、组合、评估和复用的 Agent 能力单元。  
**前沿表达**：这是企业从“功能数字化”走向“劳动力数字化”的第一步。  
**实际落地**：销售跟进 Agent、招聘筛选 Agent、客服处理 Agent、项目交付 Agent、知识助理 Agent、数据分析 Agent。  
**本质**：把岗位能力、流程动作和知识判断，转化为可部署的智能体单元。

#### 第二层：Agent 编排与集群协同能力
**定义**：让多个 Agent 不再各自为战，而是能够围绕任务进行分工、协作、路由、接力和监督，形成企业可控的 Agent 编排网络。  
**前沿表达**：真正高级的企业 AI，不是一个超级 Agent 什么都做，而是多个专职 Agent 在清晰边界下协作。  
**实际落地**：总控 Agent 负责任务拆解，知识 Agent 查制度与案例，数据 Agent 调报表，流程 Agent 调 OA/CRM/ERP，审核 Agent 做质量检查与风险拦截，人类负责人在关键节点审批。  
**本质**：把单点智能，升级为可协同的组织智能。

#### 第三层：业务执行闭环能力
**定义**：让 Agent 从“会回答”进入“能执行、能回写、能追踪结果”的闭环运行状态，真正成为业务流程的一部分。  
**前沿表达**：真正有价值的是从 Assistive AI 走向 Operational AI。  
**实际落地**：自动查询系统数据、自动生成或更新工单、自动填写表单与触发审批、自动推送消息与分配任务、自动记录操作日志、自动回写 CRM/ERP/OA。  
**本质**：让 Agent 不只是认知层工具，而是执行层生产单元。

#### 第四层：自迭代与自优化能力
**定义**：让 Agent 系统能够基于反馈、结果、人工修正、业务指标持续优化自身表现，而不是一次性交付后静态停留。  
**前沿表达**：前沿客户关心的不是“能不能上线”，而是“上线后会不会越来越准、越来越适配真实业务”。  
**实际落地**：基于用户反馈优化 Prompt 与策略，基于历史任务结果优化路由逻辑，基于人工修正沉淀规则与知识，基于成功/失败案例更新 SOP，基于指标监控持续优化 Agent 配置。  
**进阶表达**：对高频任务进行策略微调，对任务分派做动态权重调整，对知识调用质量做自动评估，对不同类型任务形成更优执行路径。  
**本质**：让企业 AI 系统具备运营意义上的学习能力，而不是空泛的“自主进化”叙事。

#### 第五层：人机共管与治理能力
**定义**：让 Agent 集群不是黑箱运行，而是具备权限控制、审计追踪、质量评估、人工接管和风险边界。  
**前沿表达**：越懂 AI 的企业，越会关注权限、越权、审计、接管和治理体系对接。  
**实际落地**：账号与权限体系打通、操作日志与审计留痕、人工审批节点、高风险任务拦截、输出质量评估、Agent 行为监控、多环境发布与灰度测试。  
**本质**：让 Agent 从实验性工具，变成企业可托付的生产系统。

#### 第六层：企业级智能运营能力
**定义**：让企业不仅部署 Agent，更能够像管理数字化系统一样管理智能体网络的成本、效果、覆盖率和 ROI。  
**前沿表达**：高级客户最终想要的，不是买几个 Agent，而是建立 AI Operating Model。  
**实际落地**：统计已部署 Agent 数量、已闭环流程数、节省人工时、响应速度提升、任务成功率/失败率、Agent 调用频次、知识与流程沉淀价值。  
**本质**：把 AI 从项目，变成企业长期运营资产。

### 3）Agent 编排与执行 Orchestration & Execution
**内容方向**：
- 用一张运行图说明 Agent 网络如何工作。
- 强调“任务进入 → 拆解 → 路由 → 调系统/知识/流程 → 质量检查 → 人工接管 → 回写与记录”。

**建议页面要点**：
- 总控 Agent 不是唯一执行者，而是调度者。
- 多 Agent 各有边界、各司其职。
- 人在关键节点拥有审批、接管与最终责任。
- 结果可以回写系统、沉淀知识、进入监控。

### 4）治理与智能运营 Governance & Operations
**内容方向**：
- 把第五层与第六层进一步展开。
- 让客户看到这不是“能不能做”的问题，而是“能不能长期运营”的问题。

**建议内容点**：
- 权限、审计、留痕、风险拦截
- 质量评估、灰度发布、问题追踪
- 效果指标、ROI、覆盖率、调用频率
- 从试点走向全局部署的管理视角

# 5.3 案例库 Case Library

## 页面目标

案例库是网站中最重要的转化型内容中心之一。它不只是“作品展示”，而是：

- 证明则知的 Agent 已经有真实落地能力。
- 帮助潜在客户快速找到与自己相近的场景。
- 通过客户故事让案例更可信。
- 通过可体验案例降低理解门槛。
- 承接对案例感兴趣的用户，进一步联系则知。

## 案例库首页建议模块

### 1）案例总览 Overview
**内容方向**：
- 作为案例总入口。
- 展示精选案例、最新案例、可体验案例入口。
- 可预留后续筛选能力。

### 2）精选案例 Featured Cases
**内容方向**：
- 选择最具代表性的 3~6 个案例。
- 优先选“最能体现落地能力”的案例，而不是最酷炫的案例。
- 尽量覆盖不同层级能力，例如：Agent 封装、集群协同、执行闭环、治理运营。

### 3）客户故事 Customer Stories
**内容方向**：
- 从使用者视角讲案例。
- 不是只讲功能，而是讲变化。

**典型表达结构**：
- 原来怎么做
- 遇到了什么问题
- 则知如何介入
- 上线后有什么变化
- 哪些工作真正被 Agent 承接

### 4）可体验案例 Interactive Demos
**内容方向**：
- 给用户一个简单体验入口。
- 不追求做全，而追求让用户感知价值。

**建议说明语**：
- 当前体验为简化版演示，正式项目可接入企业系统、知识与流程。

### 5）案例详情页 Case Detail
每个案例详情页建议尽量按统一模板制作，便于后续扩展。

**建议字段结构**：
- 案例标题
- 客户/组织类型
- 场景类型
- 对应能力层级
- 项目背景
- 原始问题
- 则知方案
- 使用方式 / 用户故事
- 项目结果
- 相关案例
- 联系按钮 / 体验按钮

# 5.4 资源中心 Resources

## 页面目标

资源中心是长期内容积累区。首版内容不需要很多，但结构要先搭好，后续便于持续添加。

它承担三个功能：

- 表达观点
- 展示动态
- 回答问题

## 推荐模块结构

### 1）文章与观点 Articles & Insights
**内容方向**：
- 则知对企业 AI Agent 落地的理解
- 对行业趋势的观察
- 对“如何开始做 Agent”的方法性文章
- 对“企业级智能体操作能力体系”的拆解文章

### 2）新闻动态 News & Updates
**内容方向**：
- 公司进展
- 项目上线
- 招聘动态
- 里程碑事件

### 3）常见问题 FAQ
